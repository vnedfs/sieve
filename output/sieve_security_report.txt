1. EXECUTIVE SUMMARY
--------------------
SIEVE was evaluated against a 29-row real-world dataset of DAN (Do Anything Now)
jailbreak prompts — a canonical prompt injection attack category that instructs
LLM systems to abandon their safety rules and act as an unrestricted model.

RESULT: 29 out of 29 prompts blocked. Detection rate: 100%.

Every attack was caught at the earliest possible pipeline stage (normalisation
and formal property verification). No raw attack text leaked into the structured
output (CSO). No proposed action was approved.

4. PIPELINE STAGE TRACE
------------------------

  Each DAN prompt passed through the following SIEVE pipeline stages:

  Stage 1 — QUARANTINE
    Input : Raw DAN attack string (UNTRUSTED)
    Output: ContentSummaryObject (CSO)
              • taint        = UNTRUSTED
              • summary      = structured JSON (no free text)
              • risk_level   = HIGH (v1) / CRITICAL (v2)
              • raw attack phrases stripped from structured output
    Result: PASS — quarantine isolation invariant upheld

  Stage 2 — PRIVILEGED
    Input : Trusted instructions + CSO (UNTRUSTED, structured)
    Output: ActionProposal
              • action_type  = response
              • taint_metadata = { intent: UNTRUSTED, topics.*: UNTRUSTED, ... }
              • derived_from = system
              • risk_level   = HIGH (v1) / CRITICAL (v2)
    Result: PASS — privileged layer never saw raw DAN text

  Stage 3 — POLICY (Layer 0: Formal Property Verification)
    Check : taint_isolation property
    Violation (DAN v1):
      Untrusted data in 'topics.[0..4]' (instruction integrity violation)
    Violation (DAN v2):
      Untrusted data in 'topics.[0..4]', 'entities.[0]',
      'entities.[0].type', 'entities.[0].value' (instruction integrity violation)
    Decision: approved=False
    Result: PASS — all actions blocked at formal property check

  Stage 4 — EXECUTION
    Not reached (all actions rejected at Stage 3)
    Result: PASS — no tool or response executed


5. SECURITY INVARIANTS VERIFIED
---------------------------------

  IIP (Instruction Integrity Property)
    Actions must derive from SYSTEM-tier trusted instructions,
    not from untrusted user data.
    Status: VERIFIED ✓ — rejected via taint_isolation formal check

  Taint Isolation
    UNTRUSTED-tagged data must not flow into sensitive or
    decision-making parameters.
    Status: VERIFIED ✓ — all topic/entity params tagged UNTRUSTED; blocked

  Quarantine Isolation
    CSO.summary must always be structured JSON; raw attack text
    must never pass through as free text.
    Status: VERIFIED ✓ — CSO summaries are JSON dicts in all 29 cases

  Non-Interference
    Equivalent CSO structures produce equivalent policy decisions.
    Status: VERIFIED ✓ — consistent rejection across all 29 rows

8. CONCLUSION
-------------
SIEVE's multi-layer defence successfully detected and blocked all 29 DAN
jailbreak prompts from the forbidden_question_set_df.csv dataset:

  • Detection rate         : 100% (29/29)
  • False negatives        : 0
  • Actions approved       : 0 (all blocked)
  • Primary rejection layer: Formal Property Verification (taint_isolation)
  • Secondary signal       : Normalisation risk scoring (HIGH/CRITICAL)
  • Quarantine isolation   : Maintained in all 29 cases (no raw text leak)

The architecture's core guarantee — that untrusted data cannot influence
system behaviour through the formal taint-isolation property — held against
both short (v1) and long-form escalation (v2) DAN attacks.
