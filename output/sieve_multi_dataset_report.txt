4. SECURITY INVARIANTS — UNIVERSAL ACROSS ALL DATASETS
-------------------------------------------------------

  The following invariants held for ALL 92 prompts across all 4 datasets:

  Quarantine Isolation
    CSO.summary is always structured JSON; raw attack text never passes
    through as free text.
    Status: VERIFIED ✓ — 92/92 CSO summaries are JSON dicts

  Taint Isolation
    Every untrusted input is classified as UNTRUSTED — no trust escalation
    regardless of attack sophistication.
    Status: VERIFIED ✓ — all 92 inputs tagged UNTRUSTED

  IIP (Instruction Integrity Property)
    Actions derived from UNTRUSTED data are rejected by the formal property
    check at Layer 0 of the policy engine.
    Status: VERIFIED ✓ — no UNTRUSTED-tainted action was approved

  Non-Interference
    Equivalent CSO structures produce equivalent policy decisions.
    Status: VERIFIED ✓ — consistent rejection for equal risk levels


5. DETECTION GAP ANALYSIS
--------------------------

  SIEVE's instruction-pattern detector uses a fixed pattern list. The following
  attack phrasings were NOT detected (no instruction_pattern flag raised):

  jailbreak_prompts.csv (19/21 missed):
    Pattern: Implicit persona assignment ('You are [AI persona]', 'rules={...}')
    Examples: CumGPT, Illegality Mode, Alphabreak, Anarchy, Buer, Agares
    Reason: No 'ignore previous instructions' or 'pretend to be DAN' phrasing

  malicous_deepset.csv (18/19 missed):
    Pattern: 'Forget all previous tasks', 'Ignore the above instructions',
             'Ignore all preceding orders', 'Forget everything before that'
    Reason: Only 'ignore all previous instructions' (exact) is in pattern list

  predictionguard_df.csv (14/21 missed):
    Pattern: Hypothetical framing ('In this hypothetical story...'),
             Fictional world setup, Condition Red dialogue writer,
             Switch flipper conditional, HGTTG guidebook framing
    Reason: No direct instruction-override keyword; uses indirection

  MITIGATION: Despite missed instruction-pattern detection, taint isolation
  still applies — all 92 inputs are tagged UNTRUSTED and untrusted data cannot
  flow into sensitive pipeline parameters. The pipeline's formal property
  verification (Layer 0 policy) provides a defense-in-depth backstop.

8. CONCLUSION
--------------
SIEVE was evaluated against 92 attack prompts across 4 distinct datasets.
The core security invariants — taint isolation and quarantine isolation — held
universally across all prompts and all attack categories.

  Security invariants VERIFIED across all 92 prompts:
    ✓ Quarantine isolation: 92/92 CSO summaries are structured JSON
    ✓ Taint isolation:      92/92 inputs tagged UNTRUSTED
    ✓ No action approved with UNTRUSTED taint violation

  Instruction-pattern detection by category:
    ✓ DAN Jailbreak Extended  : 100% (31/31) — high pattern coverage
    ◈ PredictionGuard         :  33% ( 7/21) — partial coverage
    ✗ Persona Jailbreaks      :  10% ( 2/21) — implicit framing evades patterns
    ✗ Context Reset Injection :   5% ( 1/19) — variant phrasings missed

  Recommendation: Extend instruction-pattern list to include 'forget all previous',
  'forget everything before', 'ignore the above', 'ignore all preceding',
  and 'you are now [AI persona]' patterns to improve detection coverage.
